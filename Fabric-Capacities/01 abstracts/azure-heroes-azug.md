# Fabric Capacities: From Mechanics to Strategy

**Conference:** Azure Heroes (AZUG)
**Session Length:** 45 minutes
**Audience:** IT professionals and Azure infrastructure teams

## Abstract

Microsoft Fabric runs on capacities - Azure resources you provision in your subscription, with their own scaling behavior and cost model. If you're managing Azure environments, you might be asked to provision or monitor these. But how do they actually work? What's this "bursting" and "smoothing" you've heard about? And how do you know if your organization is overpaying?

This session gives you the understanding to navigate Fabric capacity management. We'll cover how capacities work - the consumption model, throttling mechanics, and how workloads get charged - then explore the options for monitoring and optimization. Using the Capacity Metrics App, we'll look at how to interpret what's happening and what levers you can pull.

No Fabric experience required. If you're an Azure professional who wants to understand this corner of the Microsoft data platform - whether you're supporting Fabric users, evaluating costs, or just curious what all the fuss is about - this session will get you up to speed.

## Target Audience

- Azure infrastructure administrators evaluating or managing Fabric
- IT leaders responsible for Azure resource provisioning
- Cloud architects integrating Fabric into Azure environments
- FinOps teams monitoring Fabric costs

## Key Takeaways

1. How Fabric capacities work as Azure resources: consumption model, bursting, and throttling
2. Cost management strategies for Fabric in Azure subscriptions
3. Using the Capacity Metrics App to monitor and troubleshoot
4. What IT teams need to know when supporting Fabric users
