# The Fabric Decision Web: Making Implementation Choices That Match Your Organization

**Conference:** Azure Heroes (AZUG)
**Session Length:** 45 minutes
**Audience:** IT professionals and infrastructure teams

## Abstract

Every Fabric implementation forces critical decisions: How do you organize capacity? What workspace topology makes sense? How do you balance governance with enablement? The challenge? These decisions lack universal answers, and some lock you in while others can evolve.

This session helps IT teams navigate three high-impact decision points by exploring what infrastructure patterns translate to Fabric (and which don't). Drawing on real-world examples, we'll examine common failures: treating Fabric capacities like VMs, misunderstanding workspaces as the governance boundary, and centralized control models that don't scale with self-service platforms.

You'll learn to identify where your organization sits on the Business-IT spectrum, understand which Azure infrastructure concepts apply to Fabric versus which require new mental models, and distinguish between decisions needing upfront planning (capacity topology) versus those you should iterate (governance policies). Whether integrating Fabric into existing Azure estates or starting fresh, you'll leave with a practical framework for making coherent choices that fit your organizational context.

## Target Audience

- Azure infrastructure administrators evaluating Fabric
- IT leaders managing Fabric alongside traditional Azure resources
- Cloud architects integrating Fabric into Azure environments
- Enterprise architects planning analytics platforms

## Key Takeaways

1. Framework for three critical Fabric decisions: capacity organization, workspace topology, and governance approach
2. Recognition patterns for infrastructure mental models that fail in Fabric SaaS context
3. Understanding which decisions lock you in versus which can evolve (critical for IT planning)
4. Practical criteria for IT-led versus business-led implementation patterns
